1️⃣  BIENVENIDA PERSONALIZADA
    └─ Mensajes únicas según el usuario
    └─ Ejemplo: "¡Hola Juan! Bienvenido a ConVive Assistant..."

2️⃣  CONVERSACIÓN INTELIGENTE
    ├─ El usuario envía mensajes libres
    ├─ IA procesa y responde inteligentemente
    └─ Historial scrollable con timestamps

3️⃣  BÚSQUEDA CON IA
    ├─ Usuario describe sus preferencias
    ├─ IA analiza hábitos y perfil
    └─ Recomienda compañero/a más compatible

4️⃣  RECOMENDACIÓN CON SCORE
    ├─ Avatar del usuario recomendado
    ├─ Nombre y descripción
    ├─ Score de compatibilidad: 0-100% ❤️
    └─ Card interactivo con gradiente

5️⃣  VER PERFIL COMPLETO
    ├─ Modal bottom sheet
    ├─ Información detallada del usuario
    ├─ Bio, hábitos, preferencias
    └─ Botón cerrar

6️⃣  VER UBICACIÓN EN MAPA
    ├─ Integración con MapLocationPicker
    ├─ Muestra ubicación con marcador
    ├─ Dirección del departamento
    └─ Modo lectura (no editable)

7️⃣  UI/UX MODERNA
    ├─ Burbujas de chat (user vs bot)
    ├─ Colores primarios ConVive
    ├─ Animaciones y transiciones suaves
    ├─ Indicadores de carga
    ├─ Manejo de errores amigable
    └─ Totalmente responsive

Antes de usar:

☑️ Backend mock instalado:
   pip list | findstr fastapi

☑️ Backend corriendo en puerto 8000:
   http://localhost:8000/ (en navegador)

☑️ App compilada sin errores:
   flutter run (sin errores compilation)

☑️ Botón visible en home:
   Look para icono 🤖 en header derecha

☑️ Chatbot abre correctamente:
   Presiona botón → debe abrir ChatbotScreen

☑️ Mensaje de bienvenida aparece:
   Debe haber conversación inicial

☑️ Puedes enviar mensajes:
   Input funciona, mensajes aparecen

☑️ Recibes recomendaciones:
   Envía "busco compañero" → debe recomendarse alguien

☑️ Ver Perfil abre modal:
   Presiona "Ver Perfil" en card

☑️ Ver Ubicación abre mapa:
   Presiona "Ver Ubicación" → se abre MapLocationPicker

💡 TIPS IMPORTANTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Ports y Firewalls
   └─ Si usas localhost OK ✓
   └─ Si usas IP local, asegúrate que puerta 8000 esté abierta

2. Emulador vs Dispositivo Real
   ├─ Emulador: localhost funciona directo
   └─ Real: necesitas IP local de tu PC

3. Debuggear Backend
   └─ http://localhost:8000/docs (Swagger UI)
   └─ Prueba endpoints desde ahí

4. Debuggear App
   └─ flutter logs para ver mensajes
   └─ print() statements en providers/services

5. Desconexión
   └─ Si se desconecta, revisar logs
   └─ Puede ser timeout o IP incorrecta
